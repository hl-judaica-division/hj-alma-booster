Introduction

The HJ Alma Booster (Harvard Library Judaica Division Alma Booster Chrome Extension) was created—and is in the process of further development—by Tom Wagg, Harvard '20, Littauer Judaica Library Student Assistant, for the Harvard Library Judaica Division. It was built during the course of the Harvard Judaica Division's implementation of the Alma library information system software, and was subsequently shared with other Harvard Library units. This booster enables printing bibliographic records from multiple points in Alma, reduces mouse-clicking, streamlines navigation to invoice and receiving screens, and facilitates adding of Harvard local MARC fields to bibliographic and holding records for statistics and special collections processing. The Harvard Library Judaica Division welcomes collaboration with other Alma library installations to further develop this tool.

The code for the HJ Alma Booster can be found on GitHub here. To add the current version of the HJ Alma Booster to your Chrome profile, click TO BE ADDED and contact Elizabeth Vernon of the Harvard Library Judaica Division at juddiv@fas.harvard.edu for the password.

Getting Started

Thank you for downloading the HJ Alma Booster. Let's get starting boosting your Alma experience! In order to open the popup you can either press Alt+A or click the red book icon in the top right of chrome. Once you've opened up the menu you're presented with a list of different Alma boosters as well as series of icons in the top panel. From left to right the icons are;

  • Help - Open this help page in a new tab
  • Settings - Open the settings page in a new tab
  • Close - Close the popup window

Now to navigate between boosters you can either use the mouse or keyboard. Each menu item is clickable and will take you right to the page. In addition you can use the keyboard to navigate the boosters. This can either be done with; CTRL + arrow-key, where arrow-key can be either or , or with CTRL + #, where # is the pane number you wish to open.

Feel free to head into the settings and customise your experience. After that you're all set! Open Alma in a new tab and then open the extension to try out our boosters! If you need help with a specific booster just look below!

Popup Boosters

These scripts are found on the panes of the extension popup page which can be accessed by pressing Alt+A or by pressing the red book icon

In-Page Boosters

These boosters will appear on the relevant Alma page as red buttons, there is no need for input, all information is gathered from the page so you only need to click the button.

Print

Print Record

Note: An API key with bibliographic read permissions is required for this booster.
The print record booster will appear on Record View and MD Editor pages as the red printing icon shown above. After clicking the button, the booster will find the MMS ID on the page and then perform an API request for the record. The booster will then format this XML record into a printout with up to two barcodes. There will always be a barcode in the top left showing the MMS ID. Additionally, if there is a 035 field with an OCLC number that will be place in a barcode in the upper right. This printout is then displayed with the user's default chrome printing settings. If you wish to change the format or the fields that are in the barcodes then your developers can make these changes easily.

Print Invoice

Note: An API key with bibliographic read permissions is required for this booster.
The print invoice booster will appear on Invoice Details page in the invoice lines tab with the same icon as above. After clicking the button, the booster will find the unique invoice identifier on the page and use that to pull up the invoice record with an API request. From this record, the booster will loop over every PO line and find the MMS ID attached to it. Then it will perform the Print Record booster on this MMS ID (see above) and store the outputted printout. Finally, when all records are complete it will put them in invoice order and print out the entire file. You can follow the progress of the booster with the loading page that is created from the start.

FAQs

  • How does the extension work?
    This extension works using a combination of user input through the popup and JavaScript within Alma to speed up repetitive tasks. Any time that Alma loads (while this extension is enabled), a series of scripts, styles and elements will be injected into Alma. This means that we can add new buttons, functionality and specialised look to the established system.
    When boosting tasks the extension has no more power than any human user, it is simply a lot faster and will click buttons and fill out forms in the blink of an eye. However, we hope to integrate with the Alma APIs as we develop in order to streamline the process.
  • Why do I still see loading screens? This doesn't seem to have increased the speed of Alma?
    The key point here is that the extension only has as much access as a normal user. The loading screens will be just as long and the extension cannot increase the speed of Alma.
    However, it can increase the speed that you use Alma and that is where the magic happens!
  • Will this make loading Alma slower?
    No! The files that we add to Alma on load are only a few megabytes in size, which is minimal compared to what Alma is loading, you should notice no decrease in speed.
  • How do I update the extension?
    The extension will update automatically every five hours once you have it downloaded so no need to worry about keeping up to date!
